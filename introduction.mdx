## What is whecho?

whecho (pronounced "weh-cho") is a command-line tool that sends messages to webhooks. Think of it as `echo` for the modern age - instead of just printing to your terminal, whecho sends notifications to your favorite messaging platforms.

**Don't guess when a job is finished - have it message you!** ⚓

## Why whecho?

If you've ever run a long-running script or deployment and had to constantly check if it's done, whecho is for you. Simply append `&& whecho "Job finished!"` to your command and get notified on Discord, Slack, Webex, or Teams when it completes.

```bash
# Run a long build and get notified when done
make build && whecho "Build completed successfully!"

# Monitor deployments
kubectl apply -f deployment.yaml && whecho "Deployment finished"

# Track data processing
python process_data.py && whecho "Data processing complete"
```

## Supported Platforms

whecho automatically detects your webhook URL and formats messages appropriately:

- **Discord** - Full username and content support
- **Slack** - Clean text messages
- **Webex Teams** - Markdown formatting support
- **Microsoft Teams** - Standard text notifications
- **Generic Webhooks** - Fallback support for any webhook accepting JSON with a `text` field

## Use Cases

- **DevOps**: Get notified when deployments, builds, or tests complete
- **Data Science**: Track long-running training jobs or data pipelines
- **System Administration**: Monitor cron jobs and scheduled tasks
- **Development**: Alert your team when critical scripts finish
